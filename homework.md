# Домашнее задание: твой AI-ассистент

**Дедлайн:** см. расписание курса. **Сдача после дедлайна не принимается — 0 баллов, без исключений.**

---

## Задача

Подними рабочего AI-ассистента и подключи его к Telegram. Используй PicoClaw в одной из двух конфигураций: **локальная модель через Ollama** (стандартный путь) или **API-модель** (GigaChat и др. — если нет ресурсов локально). Дай ассистенту **полезный скилл** — что-то сверх обычного болтуна.

> **Если идёшь по API-пути — требования строже.** API-модели сильнее локальных, поэтому «болтовня» и базовый скилл даются легко. См. раздел «Если используешь API-модель» в критериях оценки.

---

## Что нужно сделать

### 1. Поставить и настроить

По `student-guide.md` подними бота в Telegram. Выбери одну из двух связок:

**Локально (Ollama):**
- Ollama + модель (qwen2.5:3b или больше — **не qwen3**, у него thinking-mode ломает PicoClaw)
- PicoClaw
- Telegram-бот через BotFather
- Рабочий `~/.picoclaw/config.json` + токен Telegram в `~/.picoclaw/.security.yml`

**Через API (GigaChat / OpenAI / Anthropic / любой другой, что поддерживает PicoClaw):**
- PicoClaw настроен на API-провайдера
- Telegram-бот через BotFather
- `~/.picoclaw/config.json` + токены (API-ключ + Telegram) в `~/.picoclaw/.security.yml`

⚠️ На API-пути планка для всех критериев выше — см. «Если используешь API-модель» ниже.

### 2. Выбрать и реализовать скилл

Возьми **один** скилл (или придумай свой):

| Скилл | Описание |
|-------|----------|
| Утренний дайджест | Бот шлёт сводку — погода, календарь, заголовки новостей |
| Помощник по почте | Читает / суммирует письма или пишет ответы в твоём стиле |
| VK / соц-форвардер | Мониторит группу ВК или канал и пересылает посты в Telegram |
| Планировщик учёбы | Ведёт дедлайны и шлёт напоминания |
| Учёт расходов | Парсит «кофе 350р» и держит текущую сумму |
| Журнал тренировок | Считает подходы / повторы, выдаёт недельную статистику |
| Языковой репетитор | Гоняет тебя по словарю, поправляет грамматику |
| Код-ревьюер | Смотрит куски кода и предлагает улучшения |
| Секретарь митингов | Резюмирует расшифровки voice-to-text |
| Поиск рецептов | По продуктам из холодильника подсказывает что приготовить |

> Это примеры. Любой скилл, который использует tools (чтение/запись файлов, cron, отправка сообщений, веб-поиск и т.п.) — засчитывается.

### 3. Настроить файлы персоны

Положи в `~/.picoclaw/workspace/`:

- **`SOUL.md`** — кто твой бот: имя, характер, стиль речи, **запрещённые фразы** («Чем могу помочь», «Я — языковая модель» и т.п.), 3-5 примеров реплик.
- **`AGENT.md`** — что бот умеет: список скиллов, какие tools использует, когда действует сам, когда ждёт команды. Дай примеры диалогов для каждого скилла.
- **`USER.md`** — кто пользователь: твоё имя, как обращаться, контекст (студент, часовой пояс, предпочтения).
- (опционально) **`TOOLS.md`** — настройки tools: что включено, какие API-ключи, специфические параметры.

> **Важно:** именно эти файлы и есть «душа» бота. На Ollama-пути Modelfile с `SYSTEM` **не работает** — агент собирает системный промпт из workspace, и Modelfile SYSTEM игнорируется. В Modelfile оставляй только параметры (`num_ctx`, `temperature` и т.п.). На API-пути Modelfile не нужен — параметры модели задаются в `config.json`; SYSTEM-промпт там тоже задавать нельзя.

### 4. Снять примеры работы

Проведи минимум **3 содержательных диалога** с ботом, которые показывают скилл в деле. Скриншоты или копипаст диалогов — обязательно.

---

## Что сдавать

Один **ZIP-архив** с именем `lastname_firstname_ai.zip`:

```
lastname_firstname/
  report.md          <-- одностраничный отчёт с примерами
  SOUL.md            <-- персона
  AGENT.md           <-- что умеет
  USER.md            <-- профиль пользователя
  Modelfile          <-- только на Ollama-пути, параметры модели (без SYSTEM)
  config.json        <-- конфиг PicoClaw БЕЗ токена и API-ключей
```

> `.security.yml` и `config.json` с токеном внутри **не клади в архив**.

## Формат отчёта (report.md)

```markdown
# AI-ассистент: [Имя бота]

## Модель
- Тип: [локально через Ollama / API]
- Базовая модель: [например `qwen2.5:3b` или `GigaChat-2`]
- (для Ollama) RAM на машине: [например 16 ГБ]
- Среднее время ответа: [например ~5 секунд]

## Персона
[2-3 предложения: кто бот, какой характер]

## Скилл
- Название: [например «Утренний дайджест»]
- Что делает: [2-3 предложения]
- Какие tools использует: [например cron, read_file, message]
- Как это работает: [короткое описание flow]

## Что было непросто
[Что не заработало с первого раза? Что чинил? Что удивило?]

## Вывод
[Полезно ли? Будешь ли пользоваться? Что бы улучшил?]

## 3 примера диалогов
[скрины или копипаст]
```

## Критерии оценки

Базовая часть — 8 баллов. Финальная оценка пересчитывается в шкалу 4.5–10 (см. ниже).

| Критерий | Макс | Что значит «выполнен» |
|----------|------|-----------------------|
| Бот работает в Telegram | 2 | Бот установлен, отвечает на сообщения, в отчёте есть скрин/копия диалога с видимым именем бота. Если бот не запущен или есть ошибки авторизации — 0. |
| Персона держится и интересная | 1 | В 3+ диалогах нет шаблонных фраз («Чем могу помочь», «Я — языковая модель», «Я ИИ-ассистент»); стиль и характер заметны и согласованы между ответами. Полу-балл (0.5) — если стиль есть, но в одном из ответов проскакивает шаблон. |
| Скилл реализован и работает | 2 | Скилл использует хотя бы один tool (чтение/запись файла, cron, веб-запрос, отправка сообщения и т. п.), в диалогах виден реальный вызов tool с результатом. Если скилл сводится к промпт-инструкции без tools — 1. Если скилла нет вообще — 0. |
| Конфиги полные и корректные | 1 | В архиве есть SOUL.md, AGENT.md, USER.md, config.json и Modelfile (последний — только на Ollama-пути). Если Modelfile есть — он не содержит SYSTEM, только параметры. config.json без токенов и API-ключей. Полу-балл — если один из файлов отсутствует или есть SYSTEM в Modelfile. |
| Отчёт ясный, с реальными диалогами | 2 | report.md содержит все обязательные секции, минимум 3 разных диалога, диалоги выглядят настоящими (не сгенерированы заранее). Полу-балл — если 1–2 диалога или секции «Что было непросто» / «Вывод» формальные. |
| **Базовый итог** | **8** | |

### Бонусы (макс. +2)

- **+1** — несколько скиллов в связке (минимум 3 разных, каждый со своим tool).
- **+0.5** — автоматизация по cron / расписанию (ежедневный дайджест, напоминания).
- **+0.5** — хранение состояния между сессиями через файлы (read/write).
- **+0.5** — бот работает в групповом чате с обработкой mentions и контекста чата.
- **+0.5** — документация выше базовой: разбор дизайн-решений, что не сработало и почему, альтернативы.

Сумма бонусов ограничена +2 баллами.

### Штрафы

- **−1** — в архиве найдены секреты: `.security.yml`, `config.json` с токеном, `.env` с реальными ключами.
- **−1** — выдуманные или явно сгенерированные диалоги (то, чего не было в реальности). Честно описанная неудача — НЕ штраф.
- **−0.5** — в ZIP лежит мусор: `venv/`, `__pycache__/`, `.pyc`, `.DS_Store`, кэши моделей.

Опоздание — не штраф, а отказ в приёме работы (0 баллов; см. дедлайн в начале).

### Если используешь API-модель

API-модели (GigaChat, GPT, Claude и т. п.) гораздо сильнее локальных qwen2.5:3b. Поэтому базовое «работает + держит стиль + отвечает по теме» даётся почти бесплатно. Чтобы не получать оценку только за подключение мощного API, требования смещены вверх:

- **Персона держится** засчитывается, только если бот ведёт себя очевидно нестандартно — не как «обычный ChatGPT в Telegram». Иначе — 0.5 или 0.
- **Скилл реализован** засчитывается только при нетривиальной автоматизации с несколькими tool calls. «Прочитал файл и вывел» = 1 балл, не 2.
- **Бонус +1 «несколько скиллов в связке»** — нужно 4–5 связанных скиллов с общим состоянием, не 3 разрозненных.
- **Селективные ступени (9, 9.5, 10)** — планка ещё выше: работа должна оправдывать использование платной/мощной модели реальной пользой и нетривиальной автоматизацией, а не просто «у меня GigaChat вместо qwen».

### Шкала итоговой оценки (4.5–10)

- **10** — лучшая работа в потоке. Одному студенту.
- **9–9.5** — выдающиеся работы. 2–3 студентам в потоке.
- **8.5** — все 5 базовых критериев выполнены + есть бонусы, но работа не попала в селективный топ-3 потока.
- **8** — 8/8 базовых, без бонусов.
- **7–7.5** — всё базовое работает, один критерий с недочётами на 0.5–1 балл (скилл без tools, диалоги однотипные, персона размытая).
- **6–6.5** — базовое с недочётами суммарно на 1.5–2 балла: бот собирается с трудом, скилл слабый, отчёт минимальный.
- **5** — задача не решена в смысле ТЗ: бот не отвечает в Telegram. При критерии «Бот работает в Telegram» = 0 итог не выше 5.
- **4.5** — нечего оценивать (бот не запущен, архив пустой).

**Как считается итог:** сумма базовых баллов + бонусы − штрафы. Округление до 0.5. Минимум 4.5. Максимум 10.

**Селективные ступени (9, 9.5, 10):** работы, набравшие по формуле 9 или выше, выстраиваются по качеству. 10 получает одна лучшая работа в потоке, 9–9.5 — следующие 2–3. Остальные работы, набравшие 9+ по формуле, фиксируются на 8.5. Признаки выдающегося качества: несколько скиллов в продуманной связке, нетривиальная архитектура / автоматизация, выдающийся отчёт.

---

## Важно

- **Не клади** в архив API-ключи, пароли, токены и личные данные.
- Диалоги должны быть **настоящими** — не выдуманными. Если бот ошибся — задокументируй, что произошло. Честность ценится больше, чем красивая картинка.
- Отчёт можно сгенерировать самим ботом (если справится 😊).
