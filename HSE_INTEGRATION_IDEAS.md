# Как сделать Гришу действительно полезным студенту ВШЭ

Это не реализованный код, а карта направлений с оценкой стоимости и риска. Сгруппировано от «легко и сегодня» до «нетривиально, но wow».

## Tier 1 — без интеграций, просто доделать

Можно получить почти даром, нарастая на текущей архитектуре:

- **Импорт дедлайнов из текста**. Принять «парсни вот это» с куском силлабуса или письма — Гриша достаёт даты регулярками + LLM-fallback'ом, пакетно вызывает `deadlines.py add`. Один промпт-шаг, ноль интеграций.
- **Tg-команды-шорткаты**. PicoClaw регистрирует команды в BotFather: `/today`, `/week`, `/done`, `/add`. Меньше «разговора», больше использования с телефона на ходу.
- **Экспорт в .ics**. Скрипт `deadlines.py export ics` → файл, который улетает пользователю. Один раз — и его дедлайны живут в системном календаре.

## Tier 2 — публичные API ВШЭ

- **ruz.hse.ru (расписание)**. Публичный JSON API (`https://ruz.hse.ru/api/schedule/student/<id>?start=&finish=`). Не требует авторизации, нужен только публичный id студента. Гриша утром добавляет к дайджесту «сегодня пары: 9:00 алгоритмы, 12:00 ML», вечером — «завтра первая пара в 10:30, можно подольше».
  - Стоимость: ~50 строк Python, один CRON `at 8:50`, кэш JSON в `workspace/study/schedule.cache`.
  - Риск: низкий. API стабильное, никакие токены не утекут.
- **lk.hse.ru (LMS / SmartLMS)**. Технически есть, но требует авторизованной сессии (cookie). Можно реализовать через mini-MCP с ручным sign-in раз в N дней. См. Tier 3.
- **Telegram-каналы курсов**. Многие курсы дублируют объявления в TG. Если у тебя они уже в учётке — `userbot` (Telethon) подписан, фильтрует посты с ключевыми словами `дедлайн|deadline|due|до .* мая` и автоматически предлагает Грише добавить. Полу-автоматический бэклог.

## Tier 3 — отдельный HSE MCP-сервер

Самый «выдающаяся работа» путь. Локальный MCP-сервер `hse-mcp` (Python, stdio), который PicoClaw подхватывает через `tools.mcp.servers.hse`. Сервер предоставляет инструменты:

- `hse.schedule(date_from, date_to)` — поверх ruz API
- `hse.smartlms_assignments()` — авторизованный fetch заданий и сроков
- `hse.grades()` — текущие оценки / БРС
- `hse.email_unread(filter)` — IMAP к корпоративной почте, только заголовки про дедлайны

Сценарий: Гриша **сам** раз в день вызывает `hse.smartlms_assignments()`, диффит с `deadlines.json`, добавляет/закрывает то, что появилось/исчезло, и в утреннем дайджесте показывает diff. Пользователь больше не вводит дедлайны вручную — он только подтверждает.

Стоимость: 1-2 вечера на MVP (расписание + smartlms), плюс безопасное хранение cookie в `.security.yml`. Это уже «нетривиальная автоматизация с несколькими tool calls» из критериев API-пути.

## Tier 4 — учебная аналитика поверх лога

Когда `progress.md` накапливает 2-3 месяца сессий и закрытий — над ним можно строить простой аналитический скилл:

- **Прокрастинационный профиль**. По истории `move`/`done` сравнить декларацию и реальность: курсы, по которым ты систематически просишь перенести; время суток, когда что-то реально закрывается.
- **Recommended cadence**. Если ты по теорверу регулярно сидишь по 30 мин и закрываешь — Гриша предлагает «короткий слот по ТВ в 17:00». Никаких ML, просто скользящие средние.
- **Burnout-flag**. Три дня подряд без сессий + растущая просрочка → одно сообщение «давай сегодня просто отдых, без догоняния».

Стоимость: тривиально на текущем `progress_log.py`, добавляется как `weekly_extended`.

## Что НЕ делать

- Тащить весь Notion/Obsidian API «потому что можно». Эти системы хорошо живут на пользовательской стороне, Грише не нужно становиться вторым местом хранения.
- Делать «чат с лекциями» как RAG. Это другой продукт — репетитор, не куратор. SOUL расходится.
- Voice. На API-пути TTS/STT добавляют латентность и стоимость без явного выигрыша; пользователь в TG.

## Реалистичный ближайший roadmap

1. **Сегодня (готово)**: ручной планировщик через TG + cron-дайджесты + state в файлах.
2. **Через неделю**: импорт из текста + `/today`/`/done` команды + .ics export.
3. **Через месяц**: hse-mcp v0 с расписанием. Закрывает 80% боли «забыл что завтра».
4. **Через семестр**: hse-mcp v1 с SmartLMS + аналитика по `progress.md`.
