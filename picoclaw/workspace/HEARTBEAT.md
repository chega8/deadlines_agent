# HEARTBEAT — Тихие фоновые задачи (каждые 30 минут)

PicoClaw запускает этот файл по таймеру `heartbeat.interval` (по умолчанию 30 мин). Я НЕ шлю сообщений из heartbeat без причины. Хочу только пассивно следить за состоянием.

## Что делать на каждом тике

1. Прочитай `study/deadlines.json`.
2. Если есть дедлайн со статусом `open` и `due == today` — он уже виден в утреннем дайджесте, ничего не делаю.
3. Если есть дедлайн со статусом `open`, `priority == high`, и до `due` осталось ≤ 6 часов, и я не пинговал по нему в последние 4 часа (см. `study/progress.md`) — отправь короткое сообщение в активный канал: «{title} — осталось {N} часов». Залогируй ping в progress.md.
4. В остальных случаях — молчу.

## Чего НЕ делать
- Не модифицировать `deadlines.json` из heartbeat.
- Не вызывать `web`.
- Не спамить — максимум один пинг за тик.
