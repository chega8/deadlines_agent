# study/ — состояние учебного планировщика

Два файла, два контракта.

## `deadlines.json` — машинное состояние

```jsonc
{
  "version": 1,
  "next_id": 7,                  // следующий выдаваемый id
  "items": [
    {
      "id": "d3",                // короткий, человекочитаемый
      "title": "Реферат по философии",
      "course": "Философия",
      "due": "2026-05-25",       // YYYY-MM-DD
      "priority": "high",        // high|med|low
      "tags": ["essay"],
      "status": "open",          // open|done|dropped
      "created_at": "2026-05-22T14:03:11",
      "closed_at": null
    }
  ]
}
```

**Инвариант:** мутации этого файла идут только через `scripts/deadlines.py`. Агенту запрещено редактировать его руками.

## `progress.md` — человекочитаемый append-only лог

Две секции — `## Actions` и `## Sessions`. Append-only, парсится регуляркой в `progress_log.py weekly`.

```
## Actions
- 2026-05-22 14:03 | add  | d3 | Реферат по философии, due 2026-05-25
- 2026-05-22 17:40 | done | d1 | лаба по алгоритмам
- 2026-05-23 09:00 | ping | d3 | reminder 24h

## Sessions
- 2026-05-22 18:10 | session | теорвер | 60m | ДЗ к семинару
- 2026-05-22 21:30 | session | DL      | 90m | дописал датаспилит
```

## Восстановление картины после рестарта

`deadlines.json` — источник правды по дедлайнам. `progress.md` — источник правды по тому, что и когда происходило. Если файлы разошлись (например, ручная правка) — `deadlines.json` главнее, а в progress.md просто добавляется запись о расхождении.
